Developer Features vs. Python, C, and JavaScript

What each feature does in UL and what equivalent tooling (if any) exists elsewhere.

FeatureUniLogicPythonC / JavaScript
Dynamic Runtime (DR) Settings Per-file compiler directives control memory model, safety level, type system, integer width, and concurrency model. Conflicts between files are detected at compile time with plain-English warnings. Fixed at runtime. Python uses GC always. No per-file control. No conflict detection. Fixed at compile time. C uses manual memory always. No built-in conflict detection across translation units. / JS: Fixed. JS uses GC always. No per-file control.
Memory Safety (GC / Manual / Refcount / Arena) Four memory models selectable per file. Compiler detects cross-file boundary violations before execution. GC only (CPython uses refcount + cyclic GC). No choice. No boundary detection. Manual only. Valgrind and AddressSanitizer detect issues at runtime, not compile time. / JS: GC only. No choice. No boundary detection.
FeatureUniLogicPythonC / JavaScript
Call Stack Visualizer IDE panel showing live stack frames, function names, arguments, and current line as the program runs. No breakpoints needed. Available via debugger (pdb, VS Code debugger). Requires manual breakpoints or exception triggers. Available via gdb/lldb. Requires manual breakpoints. Not editor-integrated out of the box. / JS: Available in Chrome DevTools and VS Code debugger. Requires manual breakpoints.
Side-by-Side Transpiled Output "Show C Output" command opens the compiled C output beside the UL source in a split editor. Updates on every save. No direct equivalent. dis module shows bytecode, not a target language. Compiler Explorer (godbolt.org) shows assembly. Web-based, not editor-integrated. / JS: Babel REPL shows transpiled output online. Not editor-integrated.
FeatureUniLogicPythonC / JavaScript
Per-Function Runtime Logging Toggle Toggle logging on any function from the IDE without editing source. Compiler injects entry/exit instrumentation automatically on rebuild. cProfile and line_profiler exist but require decorators in source or command-line flags. Not toggleable from an IDE panel. gprof requires recompiling with -pg flag. No GUI toggle. Manual process. / JS: console.time() must be added manually to source. No IDE toggle.
Memory Usage Tracker Allocation and deallocation events mapped back to source lines. Shows peak memory, live allocations, and leak candidates in the IDE. memory_profiler and tracemalloc exist. Require decorators or manual instrumentation. Not IDE-integrated. Valgrind/Massif and AddressSanitizer work well but are command-line tools. Not editor-integrated. / JS: Chrome DevTools heap snapshots. Not editor-integrated. No source-line mapping.
Function Call Timer Sortable table in the IDE showing call count, min/max/average time per function. No source changes required. cProfile gives this. Requires running from command line. Output is a text table, not IDE-integrated. gprof gives call counts and time. Command line only. No IDE panel. / JS: Chrome DevTools Performance panel. Not editor-integrated. No automatic source mapping.
Flame Graph Visual call tree with width proportional to time. Clickable — selecting a frame jumps to that function in the editor. py-spy and speedscope produce flame graphs. Separate tools, not editor-integrated. perf + flamegraph.pl on Linux. Separate tool, not editor-integrated. / JS: Chrome DevTools Performance tab produces flame charts. Not editor-integrated.
FeatureUniLogicPythonC / JavaScript
DR Conflict Checker Scans all files in a project. Surfaces incompatible DR settings with plain-English explanation and highlights the conflict in the editor at the call site. No equivalent. Python has no configurable runtime model. No equivalent. C has no per-file runtime policy system. / JS: No equivalent.
Dependency Graph with DR Overlay Visual map of which files call which. DR setting mismatches shown as red edges. Clicking a node opens the file. pydeps and import-graph tools show import graphs. No runtime policy overlay. include-what-you-use and doxygen can generate dependency graphs. No runtime policy overlay. / JS: Madge and webpack-bundle-analyzer show dependency graphs. No runtime policy overlay.
Inline Dead Code Detection Functions declared but never called are flagged in the editor gutter in real time as you type. vulture detects dead code. Command-line tool, not live in-editor. Clang's -Wunused-function and PVS-Studio detect unused functions. Not live in-editor for all cases. / JS: ESLint no-unused-vars catches some cases. Not always accurate for exported functions.
FeatureUniLogicPythonC / JavaScript
Code Walkthrough (AI-powered) One command narrates the open file — what each function does, what data flows where, stepping through with line highlights. Works with any OpenAI-compatible model including local Ollama. GitHub Copilot Chat has /explain. No synchronized line highlighting or step-through. Same as Python — chat-based explanation only. No step-through or line highlighting. / JS: Same as Python.
The profiler features (call timer, memory tracker, flame graph, logging toggle) require the compiler instrumentation pass — designed but not yet built.
