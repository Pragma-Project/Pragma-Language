Pragma Language Reference

A concise guide to Pragma syntax. Source files use the .run extension.

Overview

Pragma is a statically-typed systems language that transpiles to C. It is designed to be readable to programmers familiar with Java or Python while giving full access to low-level memory operations when needed. The two layers are kept visually distinct: ordinary code uses familiar lowercase keywords; anything that crosses into manual memory management is written in ALL CAPS to signal you are no longer in safe territory.

Every program has at least one GLOBAL block (declarations) and one MAIN block (entry point). Blocks are closed with end keyword rather than braces.

Program Structure

All top-level declarations (functions, objects, constants, types) live in GLOBAL. Executable entry-point code lives in MAIN.

GLOBAL

  // global declarations go here

END GLOBAL

MAIN

  // program entry point

END MAIN
Note A file may contain multiple GLOBAL and MAIN blocks; they are concatenated in order. In practice, one of each is the norm.

Comments

Single-line comments begin with //. There are no multi-line comment delimiters.

// this entire line is a comment
int x = 10  // inline comment after code

Imports

Bring in an external module by name. The module name is an identifier, not a path string.

import Math
import Collections

Variables

Declaration syntax is type name with an optional initializer. Variables are always declared before use; there is no inference.

int x = 10
float pi = 3.14159
String msg = "hello world"
bool active = true
int uninit            // declared without value — defaults to 0

Assignment and compound assignment:

x = 42
x += 5
x -= 2
x *= 3
x /= 4
x++
x--

Constants

Constants must be initialized at declaration and cannot be reassigned. They are declared inside GLOBAL.

constant int MAX_SPEED = 100
constant float G = 9.81
constant String APP_NAME = "Pragma"

Collections

array

Fixed-size, homogeneous. Size is determined by the initializer. Element type goes in brackets.

array[int] scores = [95, 87, 72, 100]
array[float] coords     // empty array

scores[0]               // index access — zero-based

List

Dynamic, resizable. Same bracket-type syntax.

List[String] names = ["Alice", "Bob", "Carol"]
names.add("Dave")
names[1]                // "Bob"

Map

Key-value store. Both key and value types are declared.

Map[String, int] scores
Note array is stack-friendly and maps directly to a C array. List and Map are heap-allocated.

Casting

Explicit type conversion uses change(expr)->type. Truncates floats to integers (no rounding).

float f = 3.9
int n = change(f)->int   // n = 3, truncated

int raw = 65
String s = change(raw)->String

If / Else

No parentheses around the condition. Blocks close with end if. The else block closes with end else before the final end if.

if x > 100
  print "large"
else if x > 50
  print "medium"
else
  print "small"
end if

While

The condition is followed by the word do. The block closes with end do.

int count = 0
while count < 5 do
  count++
  print count
end do

For

C-style three-part loop on one line: for type var = init while condition do step. Body closes with end for.

for int i = 0 while i < 10 do i++
  print i
end for

The step expression supports ++, --, and compound assignment (i += 2).

For Each

Iterates over any collection. The loop variable is inferred from the collection element type.

List[String] names = ["Alice", "Bob"]

for each name in names
  print name
end for

Conditions (Switch)

Pattern-matches on a single expression. Single-statement cases use a colon on the same line. Multi-statement cases start on the next line with no colon. Falls through to default if no case matches. No fall-through between cases.

int code = 2

conditions code
  1: print "one"
  2: print "two"
  3
    print "three"
    print "still three"
  default: print "other"
end conditions
Note Unlike C switch, there is no fall-through. Each case is independent. No escape needed to stop one case bleeding into the next.

Loop Control

KeywordEquivalentEffect
escapebreakExit the innermost loop immediately.
skipcontinueJump to the next iteration.
while j < 10 do
  j++
  if j == 5
    escape     // break out of loop
  end if
  if j % 2 == 0
    skip       // skip even numbers
  end if
  print j
end do

Functions

Declared in GLOBAL. Parameters are typed. Return type follows returns. Omit returns for void functions. Parentheses are required even with no parameters.

function Add(int a, int b) returns int
  return a + b
end function

function Greet()       // void, no 'returns'
  print "hello"
end function

Recursive functions work normally:

function Factorial(int n) returns int
  if n <= 1
    return 1
  end if
  return n * Factorial(n - 1)
end function

Objects

Pragma's equivalent of a class. Fields are declared at the top; methods follow. Instantiate with new TypeName[args] using square brackets, not parentheses.

object Runner
  String name
  int age
  float speed = 0.0   // field default
end object

// Inheritance and interface implementation on one line:
object Cyclist inherits Runner interfaces Athlete
  int bikeType

  function GetSpeed() returns int
    return change(self.speed)->int
  end function
end object

// Instantiation — square brackets, not parentheses:
Runner r = new Runner["Jake", 24, 9.8]
print r.name
Note Pragma has single inheritance only (inherits). Multiple interface compliance is expressed with interfaces A, B, C.

Interfaces

Define a contract — method signatures with no body. An object that declares interfaces Foo must implement every method in Foo.

interface Athlete
  function Train()
  function GetSpeed() returns int
end interface

Mandates (Enumerations)

Named integer constants grouped under one name — equivalent to an enum. Members can have explicit integer values or be auto-assigned. Access a member with Name_member.

mandate Priority
  low    = 1
  medium = 2
  high   = 3
end mandate

int p = Priority_high   // p = 3

Inline form for short enumerations:

mandate Direction low = 0, high = 1 end mandate

Print

Outputs any expression to standard out followed by a newline. No format string required.

print "hello world"
print x
print Add(3, 7)
print r.name

Try / Catch

Runtime errors are caught by exception type. Multiple exception types in one catch clause are separated by |. Each catch closes with end catch; the whole block closes with end try.

try
  // code that might throw
catch(IOException err)
  print "io error"
end catch
catch(RuntimeError | ValueError err)
  print "runtime or value error"
end catch
end try

RAW Block

The RAW block is where manual memory management lives. Everything inside that does not exist at the high level — pointers, direct memory reads, address-of — is written in ALL CAPS. This is intentional: it signals to the reader that they have left the managed layer and are operating directly on memory.

RAW
  // pointer declarations, MEM access, DEREF — all go here
END RAW
Warning Code inside a RAW block bypasses safety guarantees. The transpiler does not insert bounds checks or null guards within RAW. If you are using MemProof or MemResistant signing, RAW sections are flagged and must be reviewed manually.

Pointers

Declared inside a RAW block. The type comes first, then POINTER, then the name, assigned to ADDRESS<variable>. The pointed-to variable must already be declared.

RAW
  int8 x = 42
  INT8 POINTER p = ADDRESS<x>
END RAW
OperatorMeaning
ADDRESS<x>Take the address of variable x (like &x in C)
DEREF<p>Dereference pointer p (like *p in C)
MEM<addr>Read/write at a raw numeric address
SIZE<type>Byte size of a type (like sizeof in C)

Memory Access

Read or write any memory address directly. The address can be an integer expression or a pointer variable.

RAW
  uint32 addr = 0xDEAD0000
  MEM<addr>                 // read at addr

  int val = 7
  UINT32 POINTER p = ADDRESS<val>
  DEREF<p>                   // dereference p

  int bytes = SIZE<int64>    // bytes = 8
END RAW

Bitwise Operations

Bitwise operators use the same symbols as logical operators (&, |) plus three word-form operators. The word form makes it impossible to confuse bit manipulation with logic at a glance.

int flags = 0

flags = flags | 4         // set bit 2
flags = flags & 0xFF      // mask low byte
flags = flags delta 1     // XOR bit 0
flags = flags left 2      // shift left  (flags << 2 in C)
flags = flags right 1     // shift right (flags >> 1 in C)
flags = flip flags         // bitwise NOT (~flags in C)
