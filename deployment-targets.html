Pragma Deployment Targets
What is required beyond the transpiler to deploy Pragma code on each target platform.
Python Target
Transpiler + Python stdlib mapping + concurrency bridge + pip integration.
Python Standard Library Mapping
A Pragma standard library for Python interop — py.collections, py.io, py.os etc. that map to Python's stdlib. You write the Pragma call; the transpiler outputs the Python equivalent.
// Pragma
let data = io.read("file.txt")

// transpiles to Python
data = open("file.txt").read()
GIL-Aware Threading Model
Python's threading is fundamentally constrained by the GIL. Pragma's concurrency primitives need to map to threading, multiprocessing, or asyncio depending on the use case — the transpiler selects the correct backend.
// Pragma — CPU-bound: maps to multiprocessing
@parallel
function process_batch(list items)
  // ...
end function

// I/O-bound: maps to asyncio
@async
function fetch_all(list urls)
  // ...
end function
Runtime Shim
Pragma's refcount model differs from Python's. A small bridge handles object lifecycle differences so Pragma-transpiled code behaves correctly inside a Python runtime.
# generated shim (not written by hand)
# pragma_rt.py — imported at top of output

class PragmaRef:
  def __init__(self, obj):
    self._obj = obj
  def __del__(self):
    # lifecycle hook for Pragma DR
    pragma_release(self._obj)
Package Bridge
pip and PyPI are the ecosystem. Pragma declares Python dependencies in its module manifest and imports them as if they were Pragma modules — the bridge resolves and installs them.
// pragma.module
target python
depend requests   == "2.31.0"
depend numpy      >= "1.24"

// in Pragma source
import py.requests as http
let resp = http.get(url)
Java Target
Transpiler + JVM stdlib mapping + exception model bridge + Maven/Gradle integration.
JVM Standard Library Mapping
A Pragma standard library for JVM interop — jvm.collections, jvm.io, jvm.net etc. mapping to Java's stdlib. You write the Pragma call; the transpiler expands to the full Java boilerplate.
// Pragma
let resp = net.get(url)

// transpiles to Java
HttpClient client = HttpClient.newHttpClient();
HttpRequest req = HttpRequest.newBuilder()
  .uri(URI.create(url)).build();
var resp = client.send(req, ofString());
Type System Bridge
Java's type system is nominal and explicit. Pragma's types need to map cleanly to JVM types, including handling Java generics which have no direct Pragma equivalent.
// Pragma type
type UserRecord
  string name
  int    age
end type

// JVM output
public record UserRecord(
  String name,
  int    age
) implements PragmaType {}
Checked Exception Bridge
Java forces checked exceptions up the call stack. Transpiled Pragma code either wraps everything in RuntimeException or the transpiler generates the throws declarations automatically from @throws annotations.
// Pragma
@throws(IOError)
function read_file(string path)

// JVM output
public static String readFile(String path)
  throws IOException {
  // ...
}
Build Integration
Maven and Gradle are the Java ecosystem. Pragma output slots into a standard Java build pipeline the same way a .java source file would — the transpiler emits .java files into the expected source tree.
// pragma.module
target jvm
depend com.google.guava == "32.0"
depend org.slf4j        >= "2.0"

# Gradle picks up transpiler output
# src/main/pragma/ → src/main/java/
# then normal ./gradlew build
