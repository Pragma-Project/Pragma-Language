Pragma Language Reference

Source files use the .run extension. Click ⚙ Systems Mode for low-level memory and bitwise operations.

Overview

Pragma is a statically-typed systems language that transpiles to C. It is designed to be immediately readable to programmers from any background. The guiding rule is simple: everything common and agreed-upon is lowercase; everything that is uniquely low-level or unfamiliar to non-C programmers is written in ALL CAPS.

If you see a word in caps, you are touching memory directly or doing something that does not exist in Python, Java, or most other modern languages. A simple program never has to see those words.

Every program has at least a main function. Everything else — functions, objects, constants — is declared at file scope above it.

Program Structure

Declarations live at file scope. The entry point is a function named main. No wrapper blocks required.

// file scope: constants, objects, functions declared here

constant int MAX_SIZE = 100

function helper() VOID
  // ...
end function

function main()
  // entry point
end function
Note main or MAIN are both accepted as the entry point name.

Comments

Single-line only. No multi-line comment syntax.

// full line comment
int x = 10  // inline comment

Imports

import Math
import Collections

Variables

Always type name with an optional initializer. No type inference.

int x = 10
float pi = 3.14159
string msg = "hello"
bool active = true
complex z = complex(3, 4)   // 3 + 4i
int n                       // declared without value, defaults to 0

Assignment operators:

x = 42
x += 5    x -= 2    x *= 3    x /= 4
x++        x--

Constants

Declared at file scope. Cannot be reassigned.

constant int MAX_SPEED = 100
constant string APP_NAME = "Pragma"

Collections

array

Fixed-size. Size is part of the type declaration and is required. Element count goes inside a second set of brackets. Zero-indexed everywhere.

array[int[4]] scores = [95, 87, 72, 100]
scores[0]          // 95
scores[3]          // 100

Multi-dimensional: first index is rows, second is columns.

array[string[2][4]] grid    // 2 rows, 4 columns
grid[0][0] = "a"
Note Size must be declared. array[int[]] is only valid when an initializer is present — the compiler infers the count. Without an initializer, size is required.

list

Dynamic, resizable. Bracket holds element type and optional initial capacity.

list[string[3]] names = ["Alice", "Bob", "Carol"]
names.add("Dave")
names[1]           // "Bob"

list[int[]] nums   // empty list, no capacity hint

map

Key-value store. Brackets after the type hold optional initial capacity; [] means unbounded dynamic.

map[string, int][] wordCount        // unbounded
map[string, float][10] scores      // initial capacity hint of 10

Casting

Use the type name as a function call. Truncates on float-to-int (no rounding). The transpiler handles the underlying C conversion.

float f = 3.9
int n = int(f)         // n = 3
string s = string(42)  // s = "42"
float g = float(10)    // g = 10.0
bool b = bool(1)       // b = true

If / Else

No parentheses around the condition.

if score > 200
  print "high score"
else if score > 100
  print "good score"
else
  print "keep trying"
end if

While

Condition followed by do. Closes with end do.

int count = 0
while count < 5 do
  count++
  print count
end do

For

Two forms — use whichever reads better for the situation.

// Expressive form
for int i = 0 while i < 10 do i++
  print i
end for

// C-style form (commas separate init, condition, step)
for i=0, i<10, i++
  print i
end for

The step supports ++, --, and compound assign (i += 2).

For Each

list[string[3]] names = ["Alice", "Bob", "Carol"]

for each name in names
  print name
end for

Match (no fall-through)

Pattern-matches on a single expression. No case bleeds into the next. Single-statement cases use a colon on the same line; multi-statement cases follow the colon on the next line.

int code = 2

match code
  1: print "one"
  2: print "two"
  3:
    print "three"
    print "still three"
  default: print "other"
end match

Switch (fall-through)

Like C switch — execution falls through from one case into the next unless escape is used. Use when you need that behavior explicitly.

switch code
  1:
  2:
    print "one or two"  // runs for both 1 and 2
    escape
  3:
    print "three"
    escape
end switch

Functions

Use returns type for the return type. For functions that return nothing, use VOID — or omit the return type entirely (defaults to VOID).

function add(int a, int b) returns int
  return a + b
end function

function greet(string name) returns VOID
  print "Hello, " + name
end function

function doSomething()    // VOID implied — both forms are identical
end function

Recursive functions work normally:

function factorial(int n) returns int
  if n <= 1
    return 1
  end if
  return n * factorial(n - 1)
end function

Objects

Fields at top, methods below. Instantiate with new TypeName(args). Single inheritance via inherits. Multiple interface compliance via interfaces A, B.

object Player
  string name
  int age
  float score = 0.0

  function greet() returns VOID
    print "Hi, I am " + self.name
  end function
end object

object ProPlayer inherits Player interfaces Ranked
  int rank
end object

// Instantiation — parentheses like everyone else
Player p = new Player("Alex", 25, 98.5)
print p.name
Note Single inheritance only. Multiple interface compliance is allowed.

Interfaces

Method signatures only — no body. Any object declaring interfaces Foo must implement all methods.

interface Ranked
  function getRank() returns int
  function promote() returns VOID
end interface

Mandates (Enumerations)

Named integer constants. Access a member as MandateName_member.

mandate Priority
  low    = 1
  medium = 2
  high   = 3
end mandate

int p = Priority_high    // p = 3

Print

Outputs any expression to standard out. String concatenation with +. No format strings.

print "hello world"
print "Score: " + string(score)
print add(3, 7)

Try / Catch

Multiple exception types in one catch use or.

try
  // risky code
catch(IOException err)
  print "IO error: " + string(err)
end catch
catch(RuntimeError or ValueError err)
  print "runtime error"
end catch
end try

Pragma / Systems Reference

Low-level operations. Every keyword here is in ALL CAPS — a deliberate signal that you are managing memory directly. These do not exist as accessible concepts in Python, Java, or most other modern languages.

Pointers

A pointer holds a memory address. The type of pointer tells the compiler how to interpret the bytes at that address. Declared as TYPE POINTER name.

int x = 42
INT POINTER p = ADDRESS<x>     // p holds the address of x

int8 b = 7
INT8 POINTER bp = ADDRESS<b>
Note The type before POINTER is always caps because you are working at the memory level.

ADDRESS and DEREF

PragmaC equivalentMeaning
ADDRESS<x>&xTake the address of variable x
DEREF<p>*pRead the value at the address stored in p
int val = 99
INT POINTER p = ADDRESS<val>
int copy = DEREF<p>          // copy = 99

MEM — Raw Memory Access

Read or write at any numeric memory address. Used for memory-mapped I/O and hardware register access.

UINT32 POINTER reg = ADDRESS<someVar>
MEM<reg>                    // read at address reg
MEM<0xDEAD0000>             // read at hard-coded address

Pointer Arithmetic

Offset a pointer using angle brackets. The offset is in units of the pointer's element type (not raw bytes).

INT POINTER p = ADDRESS<arr>
p<0>            // element 0   — same as DEREF<p>
p<5>            // element 5   — like *(p + 5) in C
p<i + 2>        // element i+2 — arbitrary expression
p<+3>           // advance 3 from current position

SIZE

Returns the byte size of a type. Equivalent to sizeof in C.

int bytes = SIZE<int64>    // 8
int s = SIZE<float>         // 4

nothing POINTER — Generic Pointer

A pointer with no specific type — it points to "nothing in particular." Equivalent to void* in C. Use when you need to pass a pointer whose type is determined at runtime.

nothing POINTER generic = ADDRESS<anyVar>

// Cast it to a typed pointer before use:
INT POINTER typed = INT POINTER(generic)

Bitwise Operations

All bitwise operators are in ALL CAPS — they operate at the bit level and behave differently on signed vs unsigned integers. Use uint types when doing bit manipulation to avoid sign-bit surprises.

PragmaCMeaning
BOTH1&Bitwise AND — result bit is 1 only if both input bits are 1
BOTH0!(a) && !(b) / ~(a|b)Both values are 0 / zero — bitwise NOR
EITHER1|Bitwise OR — result bit is 1 if either input bit is 1
DELTA^Bitwise XOR — result bit is 1 if the bits differ
BITFLIP x~xBitwise NOT — inverts every bit. Use on uint types.
NEGATE x-xArithmetic negation
x LEFT nx << nLeft shift — multiply by 2ⁿ
x RIGHT nx >> nRight shift — divide by 2ⁿ (logical on uint)
LEFT=<<=Left shift assign
RIGHT=>>=Right shift assign
uint32 flags = 0
flags = flags EITHER1 4       // set bit 2
flags = flags BOTH1 0xFF      // mask low byte
flags = flags DELTA 1          // toggle bit 0
flags = flags LEFT 2           // shift left 2
flags = flags RIGHT 1          // shift right 1
flags = BITFLIP flags           // invert all bits
Warning — BITFLIP on signed integers BITFLIP on a signed int flips the sign bit, producing a negative number. BITFLIP 5 gives -6, not 250. Always use uint types for bit manipulation.

Bit Fields

Declare fields with a specific number of bits using angle brackets. Fields in sequence pack into adjacent bits. Group them inside an object to control layout.

object PackedHeader
  uint is_valid <1>    // 1 bit
  uint count    <7>    // 7 bits — together = 1 byte
  uint32 value         // 4 bytes, no bit field
end object
Note Bit fields only apply inside objects. They map directly to C bit fields. The total bits for a group should align to a byte boundary (8, 16, 32, 64) for predictable layout.

pack() — Struct Packing

By default the compiler may insert padding between fields to align memory. pack() removes that padding — every field follows immediately after the previous one with no gaps. Used for network packets, file headers, and hardware register maps where byte layout must be exact.

pack(PackedHeader)    // no padding between fields
Warning Packed structs can cause misaligned memory access on some architectures, which may trigger a hardware fault or a silent performance penalty. Only use when byte-exact layout is required.
